Felhasznált technológiák bemutatása és alkalmazásuk a projekt során
1. Backend technológiák
1.1 Backend architektúra áttekintése

A backend réteg a szoftveralkalmazás gerincét képezi, felelős az üzleti logika, az adatok feldolgozása és az alkalmazás kliensoldala valamint az adatbázis közötti kommunikáció biztosításáért. A projekt során kliens–szerver architektúrát alkalmaztunk, ahol a frontend HTTP kéréseken keresztül kommunikál a szerveroldali komponensekkel.

A backend architektúra kialakításánál kiemelt szempontok voltak:

Skálázhatóság: a szerver képes kezelni a növekvő felhasználószámot és az adatforgalom növekedését.

Adatintegritás: az adatok konzisztenciája és hitelessége minden tranzakció során biztosított.

Rétegzett architektúra: az üzleti logika, adatkezelés és kommunikáció külön rétegekben valósult meg, így a karbantartás és bővítés egyszerűbb.

A backend réteg modulárisan lett felépítve, elkülönítve a kontrollereket, szolgáltatásokat és adatbázis-interfészeket, ami megkönnyítette az egységtesztelést és a kód újrafelhasználhatóságát.

1.2 MySQL adatbázis-kezelő rendszer (XAMPP környezetben)
1.2.1 Általános bemutatás

A MySQL egy nyílt forráskódú, relációs adatbázis-kezelő rendszer (RDBMS), amely az SQL (Structured Query Language) szabványra épül. A relációs modell lehetővé teszi az adatok táblákban történő szervezését, a táblák közötti kapcsolatok egyértelmű definiálását, és biztosítja az adatintegritást.

A MySQL előnyei a projekt szempontjából:

Teljesítmény: képes nagymennyiségű adat gyors kezelésére.

Skálázhatóság: horizontális és vertikális skálázás egyaránt lehetséges.

Biztonság: felhasználói jogosultságok és titkosítás támogatása.

Közösségi támogatás: aktív fejlesztői közösség, rengeteg dokumentáció és eszköz.

1.2.2 Hivatalos forrás

Cím: MySQL 8.4 Reference Manual

Szerző: Oracle Corporation

Elérés: https://dev.mysql.com/doc/refman/8.4/en/

1.2.3 Alkalmazás a projektben

A MySQL adatbázist a projektben a perzisztens adatok tárolására használtuk. A XAMPP fejlesztői környezet biztosította a helyi fejlesztéshez szükséges infrastruktúrát, beleértve a MySQL szervert és a PHPMyAdmin kezelőfelületet.

Az adatbázis tervezésekor az alábbi szempontokat követtük:

Normalizálás: csökkentettük az adatok redundanciáját, optimalizáltuk a tárolást.

Kapcsolatok: elsődleges kulcsok (Primary Key) és idegen kulcsok (Foreign Key) használata az adatok közötti referenciális integritás biztosítására.

CRUD műveletek: a backend SQL lekérdezésekkel kezeli az adatok létrehozását, lekérését, frissítését és törlését.

A MySQL erőssége, hogy strukturált és biztonságos adattárolást biztosít, így a backend logika hatékonyan működhetett.

1.3 Node.js futtatókörnyezet
1.3.1 Általános bemutatás

A Node.js egy nyílt forráskódú, JavaScript futtatókörnyezet szerveroldali alkalmazásokhoz. Az eseményvezérelt, nem blokkoló I/O modell lehetővé teszi a magas teljesítményt és az erőforrások hatékony kihasználását, különösen valós idejű webalkalmazások esetén.

Előnyei:

Egységes fejlesztési nyelv: a frontend és backend kód is JavaScript alapú.

Aszinkron feldolgozás: a szerver egyszerre több kérést képes kezelni blokkolás nélkül.

Gazdag ökoszisztéma: Node.js-hez több mint egymillió modul érhető el a npm csomagkezelőn keresztül.

1.3.2 Hivatalos forrás

Cím: Node.js Documentation

Szerző: OpenJS Foundation

Elérés: https://nodejs.org/en/docs

1.3.3 Alkalmazás a projektben

A Node.js szolgált a backend futtatókörnyezeteként, kezelve:

a frontend HTTP kéréseit,

az adatbázis-műveleteket,

a válaszok visszaküldését.

Egy Express.js szerver keretében szerveztük a REST API végpontokat, biztosítva a kliens–szerver kommunikációt. A Node.js lehetővé tette az egységes JavaScript kód használatát mindkét oldalon, csökkentve a hibák számát és javítva a fejlesztés hatékonyságát.

1.4 Backend fejlesztési és biztonsági szempontok

A backend fejlesztése során különös figyelmet fordítottunk a következőkre:

Hitelesítés és jogosultság-kezelés: JSON Web Token (JWT) alapú felhasználói hitelesítés.

Adatvédelem: titkosítás (bcrypt) a jelszavak tárolására.

Hibatűrés: hibakezelés middleware segítségével, hogy a szerver stabil maradjon.

Loggolás: minden tranzakció naplózása a hibakeresés és monitoring céljából.

Dependency injection: elősegíti a tesztelhetőséget és a moduláris kódot.

Routolás: SPA (Single Page Application) támogatása.

4. Modellezési eszközök
4.1 UML diagramok
4.1.1 Általános bemutatás

Az UML (Unified Modeling Language) szabványos modellezési nyelv, amely szoftverrendszerek tervezésére szolgál.

Típusai:

Osztálydiagramok – rendszer struktúrájának ábrázolása,

Use Case diagramok – felhasználói funkciók és interakciók,

Sequence diagramok – komponensek közötti üzenetáramlás.

4.1.2 Hivatalos forrás

Cím: UML Specification

Szerző: Object Management Group

Elérés: https://www.omg.org/spec/UML/

4.1.3 Alkalmazás a projektben

Az UML diagramok segítségével az adatbázis és rendszer szerkezeti elemeit terveztük meg,

Segítettek a backend logika és frontend komponensek kapcsolatának vizualizálásában,

Egységesítették a csapatban a kommunikációt és a fejlesztési folyamatot.

4.2 ER-modell
4.2.1 Általános bemutatás

Az ER-modell (Entity-Relationship model) az adatbázis logikai szerkezetének megtervezésére szolgál.
Elemei:

Entitások – adatbázisban tárolt objektumok,

Attribútumok – az entitások tulajdonságai,

Kapcsolatok – az entitások közötti összefüggések.

4.2.2 Hivatalos forrás

Cím: Database System Concepts

Szerzők: Silberschatz, Korth, Sudarshan

Eladás: McGraw-Hill, 2019

Elérés: https://www.mheducation.com/

4.2.3 Alkalmazás a projektben

Az ER-diagram ábrázolta az adatbázis tábláit és kapcsolatait,

Segített a MySQL adatbázis normalizált és logikus felépítésében,

Lehetővé tette a backend CRUD műveletek tervezését és tesztelését.
